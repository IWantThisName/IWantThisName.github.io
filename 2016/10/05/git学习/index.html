git学习

发表于 2016-10-05 | 分类于 git | | 热度

创建本地的git仓库

新建一个目录,在该目录下进入命令行界面,执行git init命令,将会生成一个.git子目录,该目录将成为git仓库

clone别人的git仓库

执行git clone [url] 命令,将会将该git仓库下载到本地,默认下载下来的仓库名将会是最后一个/后面的名称
可以使用git clone [url] 项目名 指定下载下来的仓库名

提交操作

使用 git add 添加需要追踪的新文件和待提交的更改， 然后使用 git status 和 git diff 查看有何改动， 最后用 git commit 将你的快照记录。这就是你要用的基本流程，绝大部分时候都是这样的。
设置当前git用户的信息：
git config –global user.name ‘’ git config –global user.email 邮箱地址

git commit -a

git commit -a 自动将在提交前将已记录、修改的文件放入缓存区

如果你觉得 git add 提交缓存的流程太过繁琐，Git 也允许你用 -a 选项跳过这一步。 基本上这句话的意思就是，为任何已有记录的文件执行 git add —— 也就是说，任何在你最近的提交中已经存在，并且之后被修改的文件。 这让你能够用更 Subversion 方式的流程，修改些文件，然后想要快照所有所做的改动的时候执行 git commit -a。 不过你仍然需要执行 git add 来添加新文件，就像 Subversion 一样。

注意，如果你不缓存改动，直接执行 git commit，Git 会直接给出 git status 命令的输出，提醒你啥也没缓存。我已将该消息中的重要部分高亮，它说没有添加需要提交的缓存。 如果你使用 -a，它会缓存并提交每个改动（不含新文件）

创建分支

git checkout -b 分支名
这个命令 创建分支并切到该分支里 而且会将之前分支里的东西都复制进去

查看提交纪录

git log –oneline 查看每一次提交写的注释
git log 查看这个分支从一开始到现在所有的操作
git log –oneline –graph 查看历史中什么时候出现了分支、合并

添加远程仓库

git remote add 别名 远程仓库地址 —–>添加远程仓库

git fetch 远程仓库别名 下载远程仓库文件
git merge 远程仓库分支时报了 refusing to merge unrelated histories 这是因为远程仓库和本地仓库没有联系
这时需要在git merge 后面加一个参数–allow-unrelated-histories

git pull 别名 相当于git fetch加上git merge

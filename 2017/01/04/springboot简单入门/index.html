Springboot简单入门

发表于 2017-01-04 | 分类于 springboot | | 热度

Spring Boot是什么

Spring框架自2003年兴起，其初衷是简化JavaEE的开发，发展到今天已经有十几年的历史了。熟悉Spring的开发者都知道，Spring能方便的集成其它的各种优秀框架。但是随着你集成的框架越多，项目的基础框架搭建就越来越麻烦，依赖管理越来越复杂，配置文件开始膨胀。曾经的轻量级框架Spring也变得越来越臃肿了。于是近几年来，如何简化Spring平台下面的web开发成为了Spring社区的共同问题。Spring Boot就是在这样的背景下诞生的。它是Spring平台上面的一种全新的开发方式，其目的就是为了简化Spring应用的环境搭建以及简化开发流程。使用Spring Boot之后，你不但不需要写各种bean的配置文件，你甚至连web.xml都不需要写了！而且，只需要简单的一句maven配置，它就可以自动导入所有依赖的jar包！

Spring Boot程序框架搭建

本文的开发环境为：eclipse 4.6 + maven 3.1 + jdk1.8

  1. 首先在eclipse里面创建一个maven工程，pom文件内容如下：
    父工程：

    <parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>1.4.4.RELEASE</version>
</parent>

    依赖：(主要是模板引擎thymeleaf + hibernate) 

    <dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-thymeleaf</artifactId>
    </dependency>

    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-jpa</artifactId>
    </dependency>

    <dependency>
        <groupId>mysql</groupId>
        <artifactId>mysql-connector-java</artifactId>

    其它内容： 

    <properties>
    <java.version>1.8</java.version>
</properties>

<groupId>cn.itheima.springboot</groupId>
<artifactId>snippets</artifactId>
<version>0.1.0</version>

<build>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>
    </plugins>
</build>
    </dependency>
</dependencies

Hibernate集成

  1. 在工程的src/main/resource下面创建一个application.properties文件，指定数据库相关的配置： 

    spring.datasource.driver-class-name=com.mysql.jdbc.Driver
spring.datasource.url=jdbc:mysql://localhost:3306/spring_boot
spring.datasource.username=root
spring.datasource.password=123

spring.jpa.show-sql = true
spring.jpa.hibernate.ddl-auto = update

spring.jpa.properties.hibernate.dialect = org.hibernate.dialect.MySQL5Dialect

  1. 创建一个实体类Product 

    package cn.itheima.springboot.snippets.entity;

import javax.persistence.Column;
import javax.persistence.Entity;
import javax.persistence.GeneratedValue;
import javax.persistence.GenerationType;
import javax.persistence.Id;
import javax.persistence.Table;

@Entity
@Table(name="product")
public class Product {
    @Id
    @GeneratedValue(strategy=GenerationType.IDENTITY)
    private Long id;

    @Column
    private String name;

    @Column
    private Double price;

    public Long getId() {
        return id;
    }

    public void setId(Long id) {
        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public Double getPrice() {
        return price;
    }

    public void setPrice(Double price) {
        this.price = price;
    }

    @Override
    public String toString() {
        return "Product [id=" + id + ", name=" + name + ", price=" + price + "]";
    }
}

  2. 创建Dao 

    package cn.itheima.springboot.snippets.dao;
import org.springframework.data.repository.CrudRepository;
import cn.itheima.springboot.snippets.entity.Product;

public interface ProductDao extends CrudRepository<Product, Long> {
}

    注意这个Dao里面没有任何代码，但是这已经够了，它已经能做增删改查了。

  3. 按照web开发的三层结构，我们再来创建一个ProductService。我们这个例子很简单，就实现一个product对象的保存。

    package cn.itheima.springboot.snippets.service;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import cn.itheima.springboot.snippets.dao.ProductDao;
import cn.itheima.springboot.snippets.entity.Product;

@Transactional
@Service
public class ProductService {
    @Autowired
    private ProductDao productDao;

    public void save(Product product) {
        productDao.save(product);
    }
}

Thymeleaf模板框架集成

现在只剩web层的代码了。SpringBoot官方不建议使用jsp，而是采用一些模板引擎比如freeMarker，thymeleaf。Thymeleaf是官方推崇的一个模板框架，本例就基于它来实现web页面。

  1. 在src/main/resources下面创建一个templates目录，创建两个html文件

    a) product_form.html, 这个文件用于输入product信息

    <!DOCTYPE HTML>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
</head>
<body>
    <h1>Form</h1>
    <form action="#" th:action="@{/add_product}" th:object="${product}" method="post">
        <p>商品名字: <input type="text" th:field="*{name}" /></p>
        <p>商品价格: <input type="text" th:field="*{price}" /></p>
        <p><input type="submit" value="提交" /> <input type="reset" value="重置" /></p>
    </form>
</body>
</html>

th:action=”@{/add\_product}”表示这个表单对应一个add_product的请求，这个请求的路径一会儿会在controller里面配置。  
th:object=”${product}”表示表单的字段会封装到一个product对象里面去，这个对象一会儿也会在controller里面设置。  
th:field=”*{name}”这个对应product对象里面的字段。  

b)product_detail.html， 这个文件用于显示最终保存的product信息  

    <!DOCTYPE HTML>
    <html xmlns:th="http://www.thymeleaf.org">
    <head>
        <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
    </head>
    <body>
        <h1 th:text="'商品名字: ' + ${product.name}" ></h1>
        <h1 th:text="'商品价格: ' + ${product.price}" ></h1>
    </body>
    </html>


th:text=”${product.name}” 表示模板会接受到一个product对象，然后显示其name字段。至于这个product对象是如何传递到页面的，一会儿在controller里面可以看到。

  1. 创建ProductController 

    package cn.itheima.springboot.snippets.controller;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.ModelAttribute;
import org.springframework.web.bind.annotation.PostMapping;

import cn.itheima.springboot.snippets.entity.Product;
import cn.itheima.springboot.snippets.service.ProductService;

@Controller
public class ProductController {
    @Autowired
    private ProductService productService;

    @GetMapping("/product_form")
    public String productForm(Model model) {
        model.addAttribute("product", new Product());
        return "product_form";
    }

    @PostMapping("/add_product")
    public String addProduct(@ModelAttribute Product product, Model model) {
        productService.save(product);

        model.addAttribute("product", product);
        return "product_detail";
    }
}

    controller里面有两个方法，分别对应浏览器的两个请求。

    a)/product_form请求，这是个get请求，它返回一个product_form字符串，对应我们之前写的product_form.html，表示请求会返回到该页面，另外由于productForm方法里面设置了一个product对象，因此form表单里面的字段会自动封装到该对象里面去。

    b)/add_product请求，这个请求对应product_form里面表单的action。当表单提交的时候把请求转发到这个方法。

    之前的页面代码里面，我们看到表单的字段会封装到product对象里面去，这个对象对应我们addProduct()方法的product参数：@ModelAttribute Product product
    addProduct()方法里面，我们调用service的save方法保存了数据。
    最终返回到product_detail.html页面。为了在该页面上显示product的详细信息，我们需要把product信息传递给html模板：
    model.addAttribute(“product”, product);

        SpringApplication.run(Application.class, args);
    }
}
